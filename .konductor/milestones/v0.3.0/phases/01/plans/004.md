# Plan 04: Config Module

## Goal
Create a config management module that reads, validates, and provides config.toml values with sensible defaults.

## Tasks

### Task 1: Write config tests

- **files:** `konductor-cli/src/config.rs`
- **action:** Create `config.rs` module. Write tests for: `read_config()` returns defaults when file missing, parses valid config.toml, handles partial config (missing sections use defaults), rejects invalid values (e.g., negative parallelism).
- **verify:** `cargo test config::` — tests exist
- **done:** Test cases define expected config behavior

### Task 2: Implement Config struct and read_config

- **files:** `konductor-cli/src/config.rs`
- **action:** Define `Config` struct with nested `GeneralConfig`, `ExecutionConfig`, `GitConfig`, `FeaturesConfig`. Implement `read_config()` that reads `.konductor/config.toml`, falls back to defaults for missing fields using serde defaults. Add `Default` impl for all config structs.
- **verify:** `cargo test config::` passes
- **done:** Config module reads and validates config.toml

### Task 3: Wire config module into CLI

- **files:** `konductor-cli/src/main.rs`, `konductor-cli/src/mcp.rs`
- **action:** Add `mod config;` to main.rs. Add a `config_get` MCP tool to mcp.rs that returns the current config as JSON. Include config summary in the `status` tool output.
- **verify:** `cargo check` passes, `cargo test` passes
- **done:** Config is accessible via MCP tools

# Plan 01: State Machine Tests and Hardening

## Goal
Add comprehensive unit tests for the state module covering all transitions, file I/O, blocker management, and edge cases. Fix any bugs discovered during testing.

## Tasks

### Task 1: Add tempfile dev-dependency

- **files:** `konductor-cli/Cargo.toml`
- **action:** Add `tempfile = "3"` under `[dev-dependencies]`
- **verify:** `cargo check` passes
- **done:** tempfile crate available for tests

### Task 2: Write state I/O tests

- **files:** `konductor-cli/src/state.rs`
- **action:** Add tests for `read_state()` (missing file error, valid file round-trip, malformed TOML error) and `write_state()` using tempfile for isolated filesystem. Refactor read/write to accept a path parameter or use a test helper that sets the state file path.
- **verify:** `cargo test state::` passes
- **done:** State file I/O is tested for success and failure paths

### Task 3: Write transition end-to-end tests

- **files:** `konductor-cli/src/state.rs`
- **action:** Add tests for `transition()` covering: valid transitions update step/status/phase correctly, invalid transitions return errors, metrics.last_activity is updated. Use tempfile-based state files.
- **verify:** `cargo test state::` passes
- **done:** All transition paths are tested

### Task 4: Write blocker management tests

- **files:** `konductor-cli/src/state.rs`
- **action:** Add tests for `add_blocker()` (adds blocker, sets status to blocked) and `resolve_blocker()` (resolves correct blocker, clears blocked status when no active blockers remain, returns error for non-existent phase blocker).
- **verify:** `cargo test state::` passes
- **done:** Blocker add/resolve lifecycle is fully tested

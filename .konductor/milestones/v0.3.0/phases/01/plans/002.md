# Plan 02: Hook Module Tests

## Goal
Add comprehensive tests for the hook module covering event deserialization, file tracking, command blocking, and edge cases.

## Tasks

### Task 1: Write HookEvent deserialization tests

- **files:** `konductor-cli/src/hook.rs`
- **action:** Add tests for `HookEvent` deserialization: valid PostToolUse event, valid PreToolUse event, missing optional fields, unknown event type, malformed JSON
- **verify:** `cargo test hook::` passes
- **done:** Event deserialization is tested for all input shapes

### Task 2: Write file tracking tests

- **files:** `konductor-cli/src/hook.rs`
- **action:** Add tests for `handle_post_tool_use()`: writes file path to tracking log, creates tracking directory if missing, skips non-write tool events, handles missing cwd/filePath gracefully. Use tempfile for isolated filesystem.
- **verify:** `cargo test hook::` passes
- **done:** File tracking behavior is verified

### Task 3: Write command blocking tests

- **files:** `konductor-cli/src/hook.rs`
- **action:** Add tests for `handle_pre_tool_use()`: blocks all destructive patterns, allows safe commands, skips non-shell tool events, handles missing command field. Expand `is_destructive()` tests to cover case-insensitivity and partial matches.
- **verify:** `cargo test hook::` passes
- **done:** Command blocking is comprehensively tested

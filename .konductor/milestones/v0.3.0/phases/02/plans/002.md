# Plan 02: CI Pipeline — Tests, Checksums, and Version Sync

## Goal
Ensure CI builds run tests before releasing, generate checksums for binary integrity, and keep version numbers in sync.

## Tasks

### Task 1: Add cargo test step to build workflow

- **files:** `.github/workflows/build-release.yml`
- **action:** Add a `cargo test` step before the `cargo build --release` step. If tests fail, the build should fail and no binary is uploaded.
- **verify:** Workflow YAML has test step before build step
- **done:** Tests gate the release build

### Task 2: Generate and upload checksum files

- **files:** `.github/workflows/build-release.yml`
- **action:** After building each binary, generate a SHA-256 checksum file (`sha256sum` on Linux, `shasum -a 256` on macOS). Upload the `.sha256` file alongside the binary using the same upload action.
- **verify:** Workflow YAML has checksum generation and upload steps
- **done:** Each release binary has a corresponding .sha256 file

### Task 3: Sync Cargo.toml version with version.txt

- **files:** `konductor-cli/Cargo.toml`
- **action:** Update `version = "0.1.0"` to `version = "0.2.1"` to match version.txt. This ensures `konductor --version` reports the correct release version.
- **verify:** `grep '^version' konductor-cli/Cargo.toml` shows 0.2.1, matches `cat version.txt`
- **done:** Version is consistent across version.txt and Cargo.toml

# Plan 01: Synthetic Test Project and E2E Test Script

## Goal
Create a synthetic test project and a bash E2E test script that drives kiro-cli with the konductor agent through the full pipeline, validating artifacts at each step.

## Tasks

### Task 1: Create synthetic project spec

- **files:** `tests/e2e/synthetic-project/spec.md`
- **action:** Write a project spec for a minimal Python CLI calculator: adds two numbers from command line args, prints result. Include clear requirements and success criteria that konductor can plan against.
- **verify:** File exists with clear project description
- **done:** Synthetic project is defined

### Task 2: Create E2E test helper library

- **files:** `tests/e2e/lib.sh`
- **action:** Create a bash helper library with functions: `konductor_chat(message)` — wraps `kiro-cli chat --agent konductor --no-interactive --trust-all-tools "message"`, `assert_file_exists(path)`, `assert_file_contains(path, pattern)`, `assert_state_step(expected_step)` — reads state.toml and checks step field, `setup_test_dir()` — creates temp dir, copies synthetic project spec, `teardown_test_dir()`, `log_step(name)` — prints step name with timing.
- **verify:** `bash -n tests/e2e/lib.sh` passes (syntax check)
- **done:** Helper library ready

### Task 3: Create E2E test runner script

- **files:** `tests/e2e/run.sh`
- **action:** Create the main test script that: (1) checks KONDUCTOR_E2E=1 or exits with skip message, (2) sets up test dir with synthetic project, (3) runs `@k-init` and validates project.md/requirements.md/roadmap.md/state.toml created, (4) runs `@k-next` (plan) and validates plans exist, (5) runs `@k-next` (execute) and validates summaries exist, (6) runs `@k-next` (verify) and validates verification.md exists, (7) reports pass/fail for each step. Each step has a timeout.
- **verify:** `bash -n tests/e2e/run.sh` passes, `KONDUCTOR_E2E=1 bash tests/e2e/run.sh` runs the full pipeline
- **done:** E2E test runs locally end-to-end

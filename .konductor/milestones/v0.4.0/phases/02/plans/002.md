# Plan 02: Benchmark Harness and Baseline

## Goal
Build a benchmark harness that wraps the E2E test with timing and metric collection, establish a baseline, and provide a comparison tool.

## Dependencies
Depends on Plan 01 for lib.sh helpers and the synthetic project.

## Tasks

### Task 1: Create benchmark script

- **files:** `tests/e2e/benchmark.sh`
- **action:** Create script that sources lib.sh, runs the same pipeline as run.sh but captures: wall-clock time per step (init, plan, exec, verify) using bash `SECONDS` or `date +%s%N`, plan count (count .md files in plans/), gap count (grep gaps in verification.md), verification status (ok or issues-found), total credits (parse from kiro-cli output). Write results to `results.toml` in the test dir. Print a summary table to stdout.
- **verify:** `bash -n tests/e2e/benchmark.sh` passes
- **done:** Benchmark harness collects metrics

### Task 2: Create baseline and comparison tool

- **files:** `tests/e2e/baseline.toml`, `tests/e2e/compare.sh`
- **action:** Create baseline.toml as a template with placeholder values (to be filled after first real run). Create compare.sh that reads baseline.toml and a results.toml, compares each metric, and flags regressions: >2x slowdown on any step, verification status changed from ok to issues-found, gap count increased. Exit 0 on pass, exit 1 on regression.
- **verify:** `bash -n tests/e2e/compare.sh` passes
- **done:** Regression detection works

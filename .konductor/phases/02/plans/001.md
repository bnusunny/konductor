# Plan 01: Install Script Hardening

## Goal
Make install.sh handle all edge cases: permissions, download validation, help text, and clear error messages.

## Tasks

### Task 1: Add --help flag and usage function

- **files:** `install.sh`
- **action:** Add a `usage()` function that prints help text (flags, examples, environment variables). Parse `--help|-h` in the argument loop and call `usage; exit 0`.
- **verify:** `bash install.sh --help` prints usage and exits 0
- **done:** Help text is available

### Task 2: Add permission and download validation

- **files:** `install.sh`
- **action:** After `mkdir -p`, check if directory was actually created and is writable. After binary download, validate the file is a real binary (check for ELF magic bytes on Linux or Mach-O on macOS using `file` command) before `chmod +x`. If validation fails, remove the file and print an error.
- **verify:** Downloading a non-existent release URL produces a clear error, not a corrupt binary
- **done:** Invalid downloads are caught and reported

### Task 3: Add checksum verification support

- **files:** `install.sh`
- **action:** After downloading the binary, attempt to download `${BINARY_NAME}.sha256` from the same release. If the checksum file exists, verify the binary against it using `sha256sum` (Linux) or `shasum -a 256` (macOS). If verification fails, remove the binary and exit with error. If no checksum file exists, print a warning and continue.
- **verify:** Script attempts checksum download and verifies when available
- **done:** Checksum verification is integrated into the install flow
